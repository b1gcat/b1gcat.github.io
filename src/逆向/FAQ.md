# FAQ

## 1.动静分析时加载地址如何保持一致

| 文件类型 | PE                                                           | ELF                                           |
| -------- | ------------------------------------------------------------ | --------------------------------------------- |
| 关闭ASLR | 将`IMAGE_OPTIONAL_HEADER\DllCharacteristic`中的`IMAGE_DLLCHARACTERISTICS_DYNAMIC`字段值去掉。<br/>即：将PE中`8140`数据改为`8100`。 | `echo 0 >/proc/sys/kernel/randomize_va_space` |

## 2.调用约定

| 约定类型       | __cdecl(规范) | pascal | stdcall                | fastcall   |
| -------------- | ------------- | ------ | ---------------------- | ---------- |
| 参数传递顺序   | 右到左        | 左到右 | 右到左                 | 寄存器和栈 |
| 平衡栈者       | 调用者        | 子程序 | 子程序                 | 子程序     |
| 允许使用vararg | 是            | 否     | 是（此时调用者平衡栈） |            |

## 3.Linux Syscall参数传递约定

| 系统架构 | 32                                                           | 64                                                           |
| -------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| 参数     | 首先将系统调用号 传入 eax，然后将参数 从左到右 依次存入 ebx，ecx，edx寄存器中，返回值存在eax寄存器 | 首先将系统调用号 传入 rax，然后将参数 从左到右 依次存入 rdi，rsi，rdx寄存器中，返回值存在rax寄存器 |
